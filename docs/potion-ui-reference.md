# Potion UI Reference

## Purpose

이 문서는 `마법약 만들기` 화면을 사용자 제공 실제 플레이 영상 기준으로 다시 분석하지 않고도 UI parity 작업을 진행할 수 있도록, 영상에서 확인된 화면/행동 규칙을 저장하는 reference 문서다.

다음 세션에서는 영상 자체를 다시 읽기 전에 이 문서를 먼저 읽는다.

## Verified Gameplay Facts

- 시작 전 안내 화면이 존재한다.
- 안내 화면은 약 `9초` 후 자동 시작된다.
- 총 문항 수는 `100`이다.
- 문항당 응답 제한시간은 `3초`다.
- 각 문항은 `빨간 약` 또는 `파란 약` 중 하나를 선택하는 구조다.
- 문항마다 정답/오답 피드백이 즉시 노출된다.
- 같은 조합이라도 실제 결과는 달라질 수 있다는 안내가 있다.
- 종료 시 별도의 완료 카드가 나타난다.

### Interaction Notes

- `마법약 만들기` 카드를 선택하면 우측 상세 패널이 열리는 장면이 `00:03` 부근에 보인다.
- 카드 선택 직후 중앙 흰 보드 영역만 어둡게 dim 처리되고, 바깥 배경은 그대로 유지된다.
- 상세 패널은 보드 내부 우측에서 슬라이드 인되는 형태로 보인다.
- 상세 패널 상단에는 `뒤로` 버튼과 `9 / 18` 페이지 인디케이터가 보인다.
- 패널 본문에는 `마법약 만들기` 제목, 우측 상단 작은 노란 아이콘 배지, 메타 정보 4개 줄이 보인다.
- 메타 정보는 프레임상 `학습능력`, `총 1 라운드`, `총 6분 소요`, `난이도 상`으로 읽힌다.
- 메타 정보 아래에는 연노랑 배경의 일러스트 카드가 있고, 하단에는 설명 문단과 CTA 버튼 2개가 배치된다.
- 하단 CTA는 좌측 어두운 버튼, 우측 민트색 버튼 구성으로 보이며 우측 버튼은 `실전 응시`로 읽힌다.

### Layout Notes

- 화면 전체 배경은 매우 밝은 회백색이다.

- 카드 하단에는 파란 진행 바와 카운트다운 문구가 있다.
- 시작 버튼은 영상 기준으로 보이지 않는다.

### Representative Timestamps

- 약 `00:48`: 문항 `7`, 남은 문항 `93`, 우측 타이머 `2`
- 약 `01:12`: 문항 `15`, 남은 문항 `85`, 우측 타이머 `3`
- 약 `01:35`: 문항 `23`, 남은 문항 `77`, 우측 타이머 `2`
- 약 `03:00`: 문항 `52`, 남은 문항 `48`
- 약 `04:00`: 문항 `73`, 남은 문항 `27`
- 약 `05:00`: 문항 `95`, 남은 문항 `5`

### Header Structure

- 좌측: 현재 문항 번호
- 중앙: `제시되는 재료의 조합으로 어떤 마법약이 제조될지 예측해 보세요.`
- 우측 첫 박스: `남은 문항 N`
- 우측 두 번째 박스: 현재 문항 잔여시간 숫자 `3 / 2 / 1`

### Timer Behavior

- 문항 상단 전체 폭에 가까운 얇은 진행 바가 있다.
- 시간이 넉넉할 때는 파란색 계열이다.
- 종료 직전에는 빨간색 계열로 바뀐다.
- 숫자 박스는 초 단위 카운트다운 역할이다.

### Main Board Layout

- 메인 게임은 중앙의 큰 흰색 카드 한 장 안에서 모두 처리된다.
- 카드 내부 중앙에는 `2 x 2` 재료 그리드가 있다.
- 비활성 슬롯은 회색 placeholder 카드다.
- 활성 재료는 연한 초록색 배경광이 있는 이미지 카드다.
- 활성 재료 카드에는 현재 구현처럼 텍스트 라벨이나 배지가 보이지 않는다.
- 원본 question frame 기준 재료 이미지는 `세로 잎 묶음`, `별 모양 민트 잎`, `가느다란 풀 다발`, `고사리처럼 갈라진 잎` 4종으로 확인된다.

### Answer Buttons

- 그리드 아래에 `파란약`, `빨간약` 버튼이 좌우 배치된다.
- 기본 상태는 흰 배경 + 얇은 컬러 테두리다.
- 선택 후 피드백 단계에서는 사용자가 누른 버튼이 채워진 상태로 남는다.
- 파란 버튼은 연한 파랑 계열, 빨간 버튼은 연한 분홍/빨강 계열이다.

## Feedback Screen

### Success State

- 배경이 연한 민트/초록 톤인 결과 카드가 중앙에 뜬다.
- 큰 문구: `예측 성공`
- 하단 문구: 실제 제조된 색상 표시
  예: `빨간약이 제조되었습니다.`
- 사용자가 고른 버튼은 하단에 채워진 상태로 유지된다.

### Failure State

- 경고성 연노랑 톤의 결과 카드가 중앙에 뜬다.
- 큰 문구: `예측 실패`
- 하단 문구: 실제 제조된 색상 표시
  예: `빨간약이 제조되었습니다.`
- 실패 상태라도 레이아웃은 성공 상태와 동일하다.

### Timeout / No Answer State

- 약 `01:47` 부근에서 시간 초과 장면이 확인된다.
- 별도 완료/결과 페이지로 넘어가지 않고 기존 question card 위에 파란 안내 오버레이가 뜬다.
- 오버레이 뒤로는 기존 재료 그리드와 `파란약` / `빨간약` 버튼이 그대로 보인다.
- 안내 문구는 아래와 같다.
  `제한 시간이 지나 응답이 저장되지 않았습니다.`
  `제한 시간 내 '파란약' 또는 '빨간약' 버튼을 클릭해 주세요.`
- 오버레이 상단에는 흰색 정보 아이콘이 있다.
- 이 상태에서는 실제 제조 색상 결과 카드가 아니라 안내성 blue alert에 가깝다.

### Layout Summary

- answered feedback는 보드 위 중앙 결과 카드 형태다.
- timeout은 파란 안내 오버레이가 보드 위에 겹쳐 뜨는 형태다.

## Visual Direction

- 전반적으로 시험용 웹앱 느낌의 평평하고 정제된 UI다.
- 현재 구현의 판타지풍 질감, 그라디언트, 구리색 포인트, 장식 배경은 영상 기준과 다르다.
- 폰트는 장식 serif보다 일반적인 산세리프에 가깝다.
- 배경은 거의 흰색에 가까운 회색 계열이다.
- 메인 카드는 둥글지만 과장되지 않고, 그림자도 약하다.

## What To Preserve In Future Sessions

- `100문항`, `3초`, `즉시 피드백`, `상단 공지 2개`, `단일 중앙 카드 레이아웃`
- 질문 카드 안에서 `헤더 + 타이머 바 + 2x2 그리드 + 답안 버튼 + 중앙 피드백 카드`가 끝나야 한다.
- 게임 중 우측 설명 패널, 통계 패널, decorative hero UI를 넣지 않는다.
